Design Review

Four ways to read the cycle in thirty seconds.

A written exploration of four directions for the public Dashboard. Each takes the same five sections — prices, cycle dial, scorecard, events — and re-ranks them around a different reading of who the page is for and how fast they want the answer.

For: EditorFormat: Wireframes & notesEffort scale: low / med / high

The brief, restated

The Dashboard is the only public utility on MacroPulse — a thirty-second answer to "where is the cycle right now?" for an audience of allocators, not traders. The cycle indicator is the page's thesis; everything else is evidence.

The current layout is competent but evidence-led: prices first, thesis second. Three of the four directions below invert that order; one keeps it but earns it back with a different visual rhythm.

What every direction must keep

Calm, not noisy. Editorial type, not ticker font. Mobile-first — first viewport on a phone is non-negotiable. Orange used once per viewport as signal, not as decoration. Static HTML, vanilla CSS, lightweight.

We treat the four cycle states (● BTC Accumulation, Alt Rotation, Take Profit, Risk Off) as the page's coordinate system.

01

The Front Page

Treat the dashboard like the front page of a research weekly: a single headline (the cycle state), a sub-deck (the one-line case), and the evidence laid out as columns of a broadsheet beneath it.

The visitor sees the cycle answer before any number. The 12 prices collapse into a single horizontal "tape" rule below the headline — the same data, a quarter of the height. The full price grid lives one scroll down.

How each section changes
Cycle indicator
Promoted to hero. State name set in display Fraunces at ~120px, italic. Four-step indicator becomes a horizontal "stations of the cycle" rail under the title — where we are, where we just were, where we're heading.
Prices
Demoted to a single-line tape above the fold. The full 12-tile grid moves below the dial as a 6×2 table — labels left-aligned, prices right-aligned, monospace columns so 24h/30d numbers line up vertically.
Scorecard
Two-column broadsheet beneath. Each indicator is a short essay block (name → value → one editorial sentence). Status conveyed by a thin coloured rule on the left edge, not a pill.
Events
Right rail "What's next" — five items max, treated as datelines: "Tue 12 May · CPI · inflation."
Visual moves

Lean into Fraunces hard. Headline sizes follow a 1.5 modular scale (16 → 24 → 36 → 54 → 81). Orange used exactly twice on the first screen: the cycle-rail active dot, and the live indicator. Everything else is off-white on dark with hairline rules at 8% opacity.

CSS hint .tape { display: grid; grid-template-columns: repeat(12, minmax(0, 1fr)); gap: 0; border-top: 1px solid var(--line); border-bottom: 1px solid var(--line); } .tape > div { padding: 10px 14px; border-right: 1px solid var(--line); font-variant-numeric: tabular-nums; } .tape > div:last-child { border-right: 0; }
Trade-offs vs. current

Better

  • The 30-second promise is met on first paint — the answer is the headline.
  • Reads like a research note. Reinforces the brand voice over Bloomberg-style density.
  • Scorecard becomes scannable prose instead of a dense table.

Worse

  • Power users who came for prices have to glance at the tape, not a grid.
  • The thesis sentence has to be excellent every week — the design demands it.
  • Vertical length increases; mobile scroll-depth grows.

Make the cycle dial the actual centre of the page — a real radial compass, not a row of four chips — and let the rest of the data be evidence orbiting it.

Left half is a circular compass, half-the-viewport tall, with the four cycle states arranged as cardinal points. A thin orange needle points at the active state; a fainter ghost-needle shows where it pointed 30 days ago — so the dial encodes direction of travel, not just current state.

How each section changes
Cycle indicator
Becomes a real radial diagram — built with conic-gradient + four absolutely-positioned labels + an SVG needle. Two needles: live (orange gradient) and 30d-ago (10% white). The narration sits to its right.
Prices
Right column shows a top-4 tape (BTC, ETH, DXY, Gold) — the four prices that actually drive the cycle read. The rest live in an expandable "+8 more" panel that opens below — `<details>` with no JS.
Scorecard
Re-ordered to mirror the compass: Tier-1 indicators (M2, ISM, DXY) → Tier-2 (yields, curve, Fed) → Crypto-specific (funding, stablecoin dom). Each tier gets its own row, separated by hairline rules.
Events
Below the fold; treated as a horizontal timeline ribbon (Mon→Sun) rather than a card list. Active day pulses orange.
Visual moves

The compass is the only thing on the page that uses the orange gradient as fill — elsewhere it's reserved for hairline accents. This makes the dial unambiguously the page's signature object. On mobile it stacks above the narration and shrinks to ~280px — still legible because there are only four labels.

CSS hint .compass { aspect-ratio: 1; background: conic-gradient(from -45deg, #16212c 0 25%, #111a23 0 50%, #16212c 0 75%, #111a23 0); border-radius: 50%; position: relative; } .compass .needle { position: absolute; left: 50%; top: 50%; width: 2px; height: 42%; background: linear-gradient(180deg, transparent, var(--orange-2)); transform-origin: top center; transform: rotate(45deg); }
Trade-offs vs. current

Better

  • Memorable. The compass is the kind of object people screenshot and share.
  • Encodes direction-of-travel, which a row of chips fundamentally can't.
  • Reuses the orange accent as a single, expressive object.

Worse

  • Highest build cost. SVG needles, conic-gradient sectors, mobile reflow.
  • Risk of looking gimmicky if the brand voice is "calm research."
  • Compass takes vertical real estate that prices used to own.

The Briefing Card

Engineer the entire dashboard to fit one viewport on a laptop, with no scroll — a single, dense, screenshottable card you'd paste into an allocator's morning email.

Three horizontal bands. Top: read + next-up. Middle: every price as a 3×4 monospace tape. Bottom: scorecard as a 2-column compact list, status reduced to a coloured square glyph (▮ supportive, ▯ neutral, ▭ headwind). Whole thing fits in 880–900px on a 13" laptop.

How each section changes
Cycle indicator
Compact: title + one-sentence rationale + the four-dot rail. Sits in the upper-left band as the loudest single block on the screen, but never larger than ~25% of viewport height.
Prices
A 3-row × 4-column "tape" — same 12 instruments, but each is a single line: ticker, price, 24h. 30d % moves to a tooltip on hover. Tabular numerals so columns align.
Scorecard
Two columns of 4 + 4 indicators. Each row is one line — name, value, trend arrow, status glyph. The long editorial sentences move to a `<details>` expanded inline, or to the methodology page.
Events
Top-right "Next" stack, four items max, datelines only. Full calendar lives behind a "See all 7 days →" link.
Visual moves

Type scale collapses — Fraunces only on the cycle state and the page title; everything else is Inter at 13–15px. Use CSS Grid grid-template-rows: auto auto 1fr with min-height: 100dvh on the wrapper to lock the briefing to one viewport on desktop. On mobile it gracefully stacks; the "single-screen" promise is a desktop one.

CSS hint .briefing { display: grid; grid-template-rows: auto auto 1fr; min-height: 100dvh; gap: 0; } .briefing > * { padding: 24px 32px; border-bottom: 1px solid var(--line); } .tape { display: grid; grid-template-columns: repeat(4, 1fr); font-variant-numeric: tabular-nums; } @media (max-width: 720px) { .briefing { min-height: 0; } }
Trade-offs vs. current

Better

  • Best information density. Allocators can take it in at a glance.
  • Genuinely screenshottable — a useful artefact, not just a webpage.
  • Print stylesheet is essentially free.

Worse

  • Visual character drops. Risks looking like a terminal — exactly what the brief warns against.
  • The editorial scorecard sentences (a strength of the current page) get hidden behind hover/details.
  • Mobile experience is a stack of compressed bands, not a designed phone screen.

The Tape & Margin

Two columns, always. A narrow live tape pinned on the left; a wide editorial margin on the right where the cycle thesis, scorecard, and events read like a long-form note. The numbers stay; the writing leads.

Left rail is a fixed-width (260px) tape, position: sticky, top of the viewport, scrolls internally. Right column is a long editorial document — the cycle block at the top, then the scorecard rendered as prose paragraphs grouped by tier, then the events as a dateline list. Reads like a research note that happens to have live numbers in the margin.

How each section changes
Cycle indicator
Headline of the right column. Same four-dot rail as today, but the orange is reduced to the active dot only — no pill outlines around the inactive states.
Prices
All 12 in a vertical sticky rail on the left. Each is two lines (price / 24h % / 30d %) separated by hairline rules. Always visible while you read. On mobile the rail collapses into a horizontal scroller above the article.
Scorecard
Stops being a table. Becomes three short prose paragraphs — Tier 1 (macro), Tier 2 (rates & liquidity), Tier 3 (crypto-internal) — each ending with the indicator names in a small monospace footnote line: "M2 ↑ ISM → DXY ↓".
Events
Below the scorecard prose, as a dateline list: bold day, indented event, one-line context. Looks like the calendar at the back of a magazine.
Visual moves

This is the most "newsletter feels like a website" direction. Right column is set at a comfortable reading measure (~62ch). Left rail uses font-variant-numeric: tabular-nums and small caps for tickers. Orange appears once: the active cycle dot.

CSS hint .shell { display: grid; grid-template-columns: 260px minmax(0, 1fr); gap: 56px; } .tape { position: sticky; top: 24px; align-self: start; max-height: calc(100dvh - 48px); overflow-y: auto; } @media (max-width: 900px) { .shell { grid-template-columns: 1fr; } .tape { position: static; max-height: none; display: flex; gap: 12px; overflow-x: auto; } }
Trade-offs vs. current

Better

  • Strongest brand fit. Reads like the newsletter, not like a tool.
  • Live prices stay on screen the entire time — solves "must be visible in first viewport" without making them the headline.
  • Scorecard prose plays to MacroPulse's actual edge (the writing).

Worse

  • The mobile experience needs care — sticky rail must collapse cleanly.
  • Scorecard-as-prose is harder to keep accurate weekly than a table is.
  • Less scannable for someone who wants a number, not a story.
Recommendation

Ship The Tape & Margin, with one move borrowed from The Front Page.

The audience you describe — allocators wanting calm signal-density — does not need a louder dial or a denser terminal. They need the page to feel like the newsletter does on its best week: numbers present but not shouting, and a written read that lets them outsource thirty seconds of judgement to you. Direction 04 does that natively, and the sticky tape solves the "prices visible above the fold" constraint without making prices the headline.

Borrow one move from Direction 01: lead the right column with the cycle state set in display Fraunces — the headline-sized treatment, not the pill-sized one currently on the page. That single change is responsible for most of the "thirty-second" promise.

Hold Direction 02 (the Compass) in reserve as a possible weekly hero illustration that lives at the top of the newsletter — same idea, but as an editorial object, not a live widget. That keeps the dashboard calm and gives the gradient somewhere genuinely expressive to live.

Direction 03 (the Briefing) is the right answer if the dashboard's job is data utility. It isn't. The dashboard's job is to be the cheapest, most legible expression of the newsletter's thesis. 04 wins on that brief.
